Section 1

Introduction

Creator Assistant bridges your Gumroad®, itch.io, Jinxxy™, Lemon Squeezy®, Payhip®, or VRChat® catalog with your Discord® server, automatically granting roles when buyers verify a purchase. Setup is dashboard-first, reviewable, and built to handle both brand-new servers and migrations from older bots.

1

Install

Add the bot to your server

2

Connect

Link Gumroad®, itch.io, Jinxxy™, Lemon Squeezy®, Payhip®, or VRChat®

3

Verify

Buyers get roles instantly

For Creators

  • Automatic role assignment on purchase
  • Connect Gumroad®, itch.io, Jinxxy™, Lemon Squeezy®, Payhip®, or VRChat®
  • Map any product to any Discord® role
  • Review setup and migration plans before applying changes
  • Invite collaborating creators
  • Gate download channels to buyers
  • Monitor analytics and fraud

For Buyers

  • Click the Verify button in-server
  • Sign in with Gumroad® OAuth
  • Enter a license key from receipt
  • Prove a role in another server
  • Roles assigned in seconds
  • Re-sync with /creator refresh

Section 2

Getting Started

Install the bot, connect your store, review the recommended setup plan, and decide how verification should work in your server; usually takes under five minutes.

You need Administrator permission in your Discord® server to run the setup flow.

Prerequisites

✓ Administrator permissions in your Discord® server
✓ A Gumroad®, itch.io, Lemon Squeezy®, Payhip®, or Jinxxy™ catalog with products, or VRChat® avatars to verify
✓ Existing Discord® roles to adopt, or permission to let setup create missing ones

Setup Steps

1

Install the bot

Authorize Creator Assistant in your server. Ensure the bot's role is above all roles it needs to assign.

2

Open the secure setup flow

Run /creator-admin setup start. The bot replies privately with a secure, time-limited Open Setup Flow button. That link carries your server setup context and takes you straight into the dedicated setup page for your server.

/creator-admin setup start
3

Connect your storefronts

The setup flow starts by connecting the platforms you sell through. YUCP uses those connections to sync products, inspect your Discord® server, and prepare a recommended plan.

Connect Gumroad® (OAuth), Jinxxy™, Lemon Squeezy® (API Token), Payhip® (API Key), itch.io (OAuth), or VRChat® from the same page. Guided providers are called out when they need a one-time credential handoff before the job can continue.

4

Let the setup page orient you

The first setup screen tells you whether this server is brand new, already in progress, already set up, or needs one fix. New servers get a simple start page. Existing servers land on a maintenance view instead of a restart-looking wizard.

5

Apply the recommended setup

Click Apply Recommended Setup to queue the durable setup job. YUCP creates missing roles, writes role rules, and can reuse your existing verification message if you choose that. It does not create a verification channel automatically.

6

Open migration tools only when you are switching bots

Migration is hidden by default. Only open the migration tools if you are replacing another verification bot and need help adopting existing roles or bridging the change carefully. Migration analyzes live products it can read from connected catalog providers. Manual URL or ID mappings can still be added separately in the bot.

Section 3

Buyer Verification

Buyers verify their purchase directly in Discord® using the method that matches the product you mapped. The bot never asks for payment details.

Gumroad® OAuth

Buyer clicks Sign in with Gumroad®, authorizes read-only access to their purchase history. The bot matches purchases to your product mappings and grants roles automatically.

Lemon Squeezy®

Connect your Lemon Squeezy® store with an API token. Buyers verify using a license key from their receipt or via account lookup; the Assistant validates licenses in real time and grants roles automatically.

Payhip®

Connect your Payhip® store with an API key. Buyers verify using a license key from their receipt; the Assistant validates licenses in real time via the Payhip® API and grants roles automatically.

License Key

Buyer selects a product from a dropdown and enters the license key from their receipt. Validated in real time against your Jinxxy™ store or manual license list.

VRChat® Avatar

Buyer signs in with their VRChat® account. The system verifies they own the licensed avatar and grants the role. Supports 2FA and credential-based login.

Discord® Role

If cross-server verification is enabled, buyers prove they hold a specific role in another Discord® server. The bot checks membership via the Discord® API automatically.

Section 6

Collaborator Invites

Invite other Jinxxy™ creators to share their store with your Discord® server. Their buyers can verify and get roles without either creator exchanging credentials.

Server Owner (you)

  • Run /creator-admin collab invite
  • Bot generates a 7-day link
  • Share the link with the collaborator
  • Track connections via collab list

Collaborator

  • Open the invite link in a browser
  • Authorize with Discord® OAuth
  • Enter their Jinxxy™ API key
  • Test the webhook connection
1

Generate an invite link

Run /creator-admin collab invite. The bot replies privately with a unique link expiring in 7 days.

2

Collaborator opens the link

They visit the link and authorize with Discord® OAuth to confirm identity.

3

Collaborator enters their Jinxxy™ API key

They paste their own Jinxxy™ API key. The bot tests the connection and confirms the webhook is live. Alternatively, you can add them manually with /creator-admin collab add.

4

Collaboration is live

Buyers from the collaborator's store can now verify in your server. Manage connections with /creator-admin collab list.

Neither creator sees the other's API key or credentials. The collaborator's key is stored securely and only used to validate their buyers' license codes.

Section 7

Cross-Server Verification

Let buyers verify by proving they hold a role in a different Discord® server. Useful for partnerships, community tiers, or cross-server access passes.

Enable it

/creator-admin settings cross-serverToggles the feature on or off. Disabled by default.

Add a Discord® Role product

Use /creator-admin product add and select Discord® Role as the type. You'll specify:

  • •Source Server ID: the server the buyer must be a member of
  • •Required Role ID: the specific role they must hold in that server
  • •Granted Role: the role to assign in your server upon verification
The buyer must be a member of the source server at the time of verification. If they later leave, /creator refresh may revoke the role depending on your role-sync settings.

Section 8

Liened Downloads

Liened downloads are Discord® channels where access is tied to verified buyers only—like a lien secures a claim on property. Only members who have verified a purchase can view the channel, read messages, or access files and links. Unverified members see the channel in the list but cannot read its contents; they're prompted to verify first. This keeps your assets exclusive, reduces piracy, and ensures buyers get a clean, gated experience without manual checks.

1

Go to the channel to protect

Navigate to the channel; for example #downloads or #exclusive-files.

2

Run the setup command

Run /creator-admin downloads setup in that channel. The bot adjusts permissions so only the verified role can read messages.

3

Post your files

Upload files or paste download links. Unverified members attempting access are redirected to the verification panel.

Make sure the bot's role is positioned above the verified buyer role in Server Settings → Roles, otherwise Discord® will block permission changes.

Section 9

Fraud Protection & Moderation

Built-in tools to identify and block fraudulent verification, automatically via OAuth validation, and manually when you need finer control.

Automatic protection

OAuth-based verification (Gumroad®, Discord®) means the buyer must actually own the account; fake keys or screenshots won't work. License key validation (Jinxxy™, Lemon Squeezy®, Payhip®) hits the provider's API in real time, so revoked or reused keys are rejected immediately.

Suspicious flag

  • Blocks all future verification attempts
  • User sees a "contact moderator" message
  • Existing roles preserved until unverified
  • Visible in moderation list with reason

Unverify

  • Revokes a specific product's role instantly
  • Audit trail preserved
  • User can re-verify with a valid key
  • Use after a chargeback or license revoke

Section 11

Manual License Keys

Issue custom license keys that are not tied to any storefront. Perfect for off-platform sales, giveaways, beta access, or Patreon rewards.

Key Lifecycle

●

Active

Key is valid and can be redeemed

●

Exhausted

Max uses reached

●

Expired

Past expiration date

●

Revoked

Manually disabled by creator

How It Works

1

Create keys

Generate keys from the dashboard or via the Public API. Each key can have an optional max-use limit and expiration date.

2

Distribute to buyers

Share keys through email, DMs, Patreon, or any channel you prefer. Bulk import is supported for large batches.

3

Buyer verifies

The buyer enters the key in the verification flow, just like a storefront license key. The bot validates it and grants the mapped role.

4

Lifecycle managed automatically

Keys are tracked for usage count and expiration. Exhausted or expired keys are rejected automatically. You can revoke a key at any time from the dashboard.

Security

  • Keys are stored hashed, never in plaintext
  • One-way hash means keys cannot be recovered
  • Revocation takes effect immediately
  • Usage attempts are logged in the audit trail

Management

  • Create individual keys or bulk import
  • Set optional max uses per key
  • Set optional expiration dates
  • Manage via dashboard or Public API
Manual license keys appear as License Key Only in the product type selector when adding a product via /creator-admin product add.

Section 12

Outgoing Webhooks

Subscribe to events and receive HTTP callbacks whenever something important happens in your verification system. Build custom integrations, sync with external tools, or trigger workflows.

Supported Events

Verification Completed
Fires when a buyer successfully completes verification
Entitlement Granted / Revoked
Fires when a role entitlement is granted or revoked
Role Synced
Fires when Discord roles are synced after a refresh
Refund Processed
Fires when a refund is processed and roles are adjusted

Security

Every webhook payload is signed with HMAC-SHA256. The signature is included in the X-Signature-256 header so your endpoint can verify that the request is authentic and has not been tampered with.

Retry Policy

If your endpoint returns a non-2xx status code or times out, the system retries with exponential backoff:

30s → 5 min → 30 min → 2 hours → 8 hours

After all retries are exhausted, the delivery is moved to a dead letter queue. You can inspect and replay failed deliveries from the dashboard.

Management

Dashboard

  • Create and manage webhook subscriptions
  • View delivery history and status
  • Inspect and replay failed deliveries
  • Rotate signing secrets

Public API

  • Programmatically manage subscriptions
  • Query delivery logs
  • Replay failed deliveries via API
  • Full CRUD on webhook endpoints

Section 13

API Access

Programmatically manage your verification data, products, entitlements, and more through the Public API v2 with 56 endpoints.

Getting an API Key

1

Open Account Settings

Navigate to Account → API Keys in the Creator Assistant dashboard.

2

Generate a new key

Click Generate API Key. The key is shown once; copy it immediately and store it securely.

3

Authenticate requests

Include the key in the Authorization header: Bearer ypsk_...

Key Format

ypsk_••••••••••••••••••••••••••••••••

All API keys start with the ypsk_ prefix. The full API covers 56 endpoints for managing verification data, products, entitlements, webhooks, and more.

Treat your API key like a password. Do not commit it to source control or share it publicly. You can revoke and regenerate keys at any time from the dashboard.

Section 14

Audit Log

Every significant action is logged with full context: who did it, when, and what changed. Use audit logs for compliance, debugging, and support investigations.

What Gets Logged

Verifications
Successful and failed verification attempts with method and product
Role Changes
Roles granted, revoked, or synced with timestamps and reasons
Refunds
Refund detections and any resulting role revocations
Settings
Configuration changes: product mappings, connections, server settings

Viewing Logs

Open Dashboard → Audit Logs to browse the full event history. Use the built-in filters to narrow results:

→ Filter by event type (verification, role change, refund, settings)
→ Filter by user (Discord ID or username)
→ Filter by date range
Audit logs are also queryable via the Public API, making it easy to integrate with external compliance or monitoring tools.
